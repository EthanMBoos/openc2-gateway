# OpenC2 Gateway Glossary

Quick reference for key terms used throughout the gateway documentation. Understanding these is essential for implementing features correctly.

---

## Protocol & Wire Format

| Term | Definition |
|------|------------|
| **Frame** | A single JSON message on the WebSocket. Has `protocolVersion`, `type`, `vehicleId`, `timestampMs`, `gatewayTimestampMs`, and `data` fields. |
| **Envelope** | The protobuf wrapper (`VehicleMessage`) that discriminates message types on the vehicle⟷gateway link. |
| **Payload** | The `data` field contents within a frame — varies by message type. |

---

## Timestamps & Ordering

| Term | Definition |
|------|------------|
| **timestampMs** | Vehicle timestamp (milliseconds since Unix epoch). **UNTRUSTED** — vehicles lack RTC/NTP. Use for display only. |
| **gatewayTimestampMs** | Gateway timestamp. Added by gateway when translating to JSON. **Authoritative** for cross-vehicle correlation. |
| **seq** | Sequence number. Monotonic counter per vehicle, wraps at 2³². The **only reliable** way to order telemetry. |
| **HWM** | High-water mark. The last accepted `seq` for a vehicle. Used for deduplication — `seq ≤ HWM` is dropped as stale. |

> ⚠️ **Never filter or reject telemetry based on `timestampMs`.** Vehicle clocks drift hours or days. Use `seq` for ordering, `gatewayTimestampMs` for correlation.

---

## Vehicle State

| Term | Definition |
|------|------------|
| **ONLINE** | Vehicle is actively sending telemetry. |
| **STANDBY** | No telemetry for `OPENC2_STANDBY_TIMEOUT` (default 3s). May be idle or briefly disconnected. |
| **OFFLINE** | No telemetry for `OPENC2_OFFLINE_TIMEOUT` (default 10s). Assumed lost or powered down. |
| **Heartbeat** | Low-frequency keep-alive from vehicles (1Hz) to maintain ONLINE status when not sending telemetry. |
| **Capabilities** | What a vehicle supports: commands (`goto`, `stop`), extensions, missions, sensors. Advertised via heartbeat. |

---

## Extensions

| Term | Definition |
|------|------------|
| **Namespace** | Unique identifier for an extension (e.g., `husky`, `camera`). Maps to a registered codec. |
| **Codec** | Go interface that encodes/decodes extension payloads. Translates protobuf ⟷ JSON. |
| **Manifest** | YAML file declaring an extension's commands and UI metadata. Lives alongside the codec. |
| **ExtensionData** | Protobuf wrapper containing `version` + `payload` bytes. Enables schema evolution per-extension. |

---

## Commands

| Term | Definition |
|------|------------|
| **commandId** | UUID generated by UI to correlate commands with ACKs. Must be unique per command instance. |
| **ACK** | Acknowledgment frame from gateway (or vehicle) confirming command receipt/execution. |
| **Rate limit** | Max 10 commands/sec per vehicle (configurable). Protects radio and vehicle processor. |
| **Idempotent** | Commands should be safe to retry. Vehicle deduplicates by `commandId`. |

---

## Transport

| Term | Definition |
|------|------------|
| **UDP multicast** | Vehicle⟷gateway transport. Zero-config, tolerates packet loss. Address: `239.255.0.1:14550` (in) / `239.255.0.2:14551` (out). |
| **WebSocket** | Gateway⟷UI transport. Reliable, ordered, full-duplex over TCP. Default: `ws://localhost:9000`. |
| **Datagram** | Single UDP packet. Must be <1400 bytes to avoid fragmentation. |

---

## Architecture

| Term | Definition |
|------|------------|
| **Gateway** | Go process that bridges vehicles (protobuf/UDP) to UIs (JSON/WebSocket). |
| **Registry** | In-memory store of known vehicles and their status. Source of truth for fleet state. |
| **Router** | Validates and forwards commands from UI to vehicles. Enforces capabilities and rate limits. |
| **Tracker** | Tracks in-flight commands, matches ACKs, handles timeouts. |

---

## Naming Conventions

| Context | Convention | Example |
|---------|------------|---------|
| Protobuf fields | `snake_case` | `sequence_num`, `supported_extensions` |
| JSON fields | `camelCase` | `sequenceNum`, `supportedExtensions` |
| Go types | `PascalCase` | `VehicleTelemetry`, `SequenceTracker` |
| Environment vars | `SCREAMING_SNAKE_CASE` | `OPENC2_WS_PORT`, `OPENC2_STANDBY_TIMEOUT` |

The gateway translates automatically between protobuf `snake_case` and JSON `camelCase`.
